# UI Copywriting & Tone Guidelines

Rules for generating text presented to the end user in interfaces.

## Casing

- Use **Sentence case** for titles, headings, and buttons (e.g., "Save changes"
  instead of "Save Changes").

## Tone & Voice

- Keep the language active, empathetic, and professional.
- Avoid robotic, dense, or overly academic phrasing.
- Be concise. Remove filler words safely.

## Error Messages

- Great error messages explain 3 things:
  1. What happened?
  2. Why did it happen?
  3. How can the user fix it?
- Bad: "Invalid input."
- Good: "We couldn't save your profile because the email is in use. Please try
  another email or log in."
