# Skill: Markdown Mastery

Guidelines for creating professional, highly readable, and semantic
documentation using Markdown.

## 1. Core Principles

- **Readability First:** Use clear formatting to make the document easily
  scannable.
- **Semantic Structure:** Use appropriate heading levels (`#`, `##`, `###`) to
  create a logical table of contents.
- **Consistency:** Follow a uniform style for listings, code blocks, and
  emphasis across all project documentation.

## 2. Technical Standards

- **GitHub Flavored Markdown (GFM):** Support tables, task lists (`- [ ]`), and
  syntax highlighting in fenced code blocks.
- **Relative Linking:** Use relative paths for internal cross-referencing
  between files in the repository.
- **Media Embedding:** Use the standard syntax `![Alt Text](path/to/image.png)`
  for images and ensure they are optimized for web.
- **Escaping:** Correctly escape special characters (e.g., `_`, `*`, `[`) when
  they are intended to be literal rather than formatting cues.

## 3. Best Practices

- **Alert Blocks:** Use callouts (e.g., `> [!NOTE]`) to highlight critical
  information or tips.
- **Code Snippets:** Always specify the language for syntax highlighting (e.g.,
  ` ```typescript `).
- **Frontmatter:** Include YAML frontmatter (`---`) for metadata and
  instructions in workflow or system files.
- **Length Control:** Keep paragraphs short and use lists to break up dense
  sections of text.
